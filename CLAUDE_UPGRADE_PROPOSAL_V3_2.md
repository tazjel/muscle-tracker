- **Task 2.1**: Develop a **Video-to-Splat Pipeline**. Instead of static photos, users record a 5-second "slow rotate" video. 
- **Task 2.2**: Implement **Blur-Aware Keyframe Selection**. The engine automatically selects the 3 sharpest, most perpendicular frames (Front, Side, Side-Angle) from the video stream to feed the Volumetric models.

### 2.3 — G15: Delta-Only State Reporting
- **Task 3.1**: Optimize Mobile-to-Cloud sync. Instead of re-uploading full 15MB images, the engine calculates the **Visual Delta** (the muscle region only) and uploads the cropped, high-resolution ROI + metadata.
- **Task 3.2**: Implement **Binary State Diffing** for the `symmetry_assessment` table to reduce database overhead.
